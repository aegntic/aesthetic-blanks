# aesthetic-blanks

Isolated, independent aesthetic website templates generated by the **website-parts-cloner** skill.

Each template is a complete, self-contained Next.js App Router project that:

- Strips all original brand assets and copy
- Converts every image/video into a drag-and-drop / click-to-upload `PartImage`
- Converts every text block into an agent-ready prompt that preserves original nuance, chunk size, hierarchy and tone
- Reverse-engineers motion (GSAP-style staggers, parallax intent, etc.) into editable `PartMotion`
- Uses a consistent `{%part:id%}` labelling system

Every site folder is **fully independent**. You can:

- Clone the whole monorepo
- Sparse-checkout a single template
- Copy a folder out and treat it as its own repo
- Push new blanks as additional top-level directories

## Pull a single template

```bash
# Option A — sparse checkout (recommended)
git clone --filter=blob:none --sparse https://github.com/aegntic/aesthetic-blanks.git
cd aesthetic-blanks
git sparse-checkout set alitwotimes
cd alitwotimes
npm install && npm run dev

# Option B — just copy the folder
# (or use gh / git archive)
```

## Add a new blank

1. Run the website-parts-cloner skill on a target URL.
2. Move the generated project into a new top-level folder:
   ```bash
   mv /path/to/parts-clone-foo ./foo
   ```
3. Commit & push. Done. The new template is immediately isolatable.

## Template conventions

- Own `package.json`, `next.config.ts`, Tailwind, etc. (zero monorepo tooling dependency)
- `PARTS_MANIFEST.json` — machine-readable map of every `{%part%}`
- `AGENTS.md` — exact instructions for any agent that wants to brand the wireframe
- `prompts/` — human-readable collection of the text prompts
- Design tokens live in `lib/design-tokens.ts`

## Studio (catalog + live edit)

```bash
cd studio && npm install --include=dev && npm run dev
# http://localhost:3410 — grid · multi-page edit · export gated
```

Live edit free; JSON export requires unlock on `/pricing` (demo unlock; Stripe later).
